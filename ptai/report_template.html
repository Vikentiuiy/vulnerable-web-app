Blind spots — ironclad proofs

Each proof ships a positive control in the same code: a sink the engine DOES flag sits beside one it misses, on the same tainted variable. That rules out a broken pipeline, an unrecognised entry point, or unreachable taint — the miss is a genuine sink blind spot.

Python SQL injection missed by PT AI

sql-app/vulns/vuln89_combo.py — one HTTP parameter, two sinks. Semgrep flags the SQLi; PT AI flags only the command injection above it.

What this says about the engine

STRONG

JVM injection taint

Java leads both tools (61%); Kotlin is PT AI-only territory — Semgrep has essentially no Kotlin taint analysis.

BLIND

Python SQLi · XSS · C memory-safety

Reproducibly missed with a working positive control beside them. Semgrep covers the first two; neither covers C well.

CONFIG

Scan the build output for TS

PT AI finds 0 on TypeScript source but 42% on the compiled JS. Semgrep reads .ts directly (58%).

N/A

Native & standalone SQL

Swift and pure .sql (schema/procs) yield 0 — outside PT AI's model. Give SQL an application taint source and it still misses the SQLi.
